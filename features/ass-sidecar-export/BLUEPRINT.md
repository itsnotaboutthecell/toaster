# Blueprint: ass sidecar export

## Architecture decisions

- **R-001 (setting + typed handler)** — follow the exact shape of
  `normalize_audio_on_export`:
  - Field declaration mirrors `src-tauri/src/settings/types.rs:268-269`.
  - Default mirrors `src-tauri/src/settings/defaults.rs:541`.
  - Setter command mirrors
    `src-tauri/src/commands/app_settings.rs:487-492`.
  - Registration mirrors `src-tauri/src/lib.rs:223`.
  - specta regenerates `src/bindings.ts`
    (`changeExportAssSidecarSetting`) automatically — no manual edit.

- **R-002 (toolbar toggle)** — mirror the button at
  `src/components/editor/EditorView.tsx:538-548`. The new button reads
  the persisted setting via the existing settings hook rather than a
  local `useState`. The `burnCaptions` local-state pattern is a
  pre-existing divergence (`EditorView.tsx:49`) and is NOT refactored
  in this bundle; keeping ephemeral burn state avoids scope creep.

- **R-003 (single-source-of-truth placement)** — the ASS document is
  authority-owned by the backend. Extract a new private helper
  `build_export_ass_doc` in `src-tauri/src/commands/waveform/commands.rs`
  (or in a small new sibling file `export_captions.rs` under
  `commands/waveform/` if keeping `commands.rs` under the 800-line cap
  demands it). The helper takes
  `(&words, &segments, &settings, frame_size)` and returns
  `Option<String>` — `None` for non-video / audio-only exports,
  `Some(doc)` otherwise. Both the burn branch and the sidecar branch
  consume the same `Option<String>`. `blocks_to_ass` is called from
  exactly **one** site across `src-tauri/src/commands/waveform/`.

- **R-004 (orthogonality)** — the burn branch and the sidecar branch
  are sibling `if` blocks inside `export_edited_media` that each
  unwrap the shared `Option<String>`. The sidecar write happens before
  the FFmpeg spawn so a write failure short-circuits without wasting
  an encode pass.

## Single-source-of-truth placement

- Caption layout authority: unchanged. Lives in
  `src-tauri/src/managers/captions/layout.rs` and serialization in
  `src-tauri/src/managers/captions/ass.rs`.
- Export-time ASS document authority: the new
  `build_export_ass_doc` in `commands/waveform/`. Both the burn branch
  and the sidecar branch are frontend-of-backend consumers of this
  one function.
- Settings authority: `AppSettings` (types.rs) + typed setter commands.
  Frontend never constructs settings rows directly.

## Data flow

```
EditorToolbar toggle  ──► changeExportAssSidecarSetting (Tauri cmd)
                               │
                               ▼
                         settings.json (persist)

Export button ──► exportEditedMedia(input, output, burnCaptions?)
                      │
                      ▼
     [commands::waveform::export_edited_media]
                      │
                      ├─► build_export_ass_doc(...) -> Option<String>
                      │         (calls blocks_to_ass ONCE)
                      │
                      ├─► if burn:     write <output>.burn_captions.ass (temp)
                      │                append -vf "subtitles=..." to ffmpeg args
                      │
                      ├─► if sidecar:  write <output>.ass
                      │                (no ffmpeg change)
                      │
                      └─► spawn ffmpeg
```

## Migration / compatibility

- Additive field with `#[serde(default)]`; legacy `settings.json`
  files load unchanged with `export_ass_sidecar_enabled = false`.
- No breaking change to `exportEditedMedia` Tauri command signature —
  the sidecar flag is read from `AppSettings` on the backend, not
  threaded through the RPC surface. This mirrors how
  `normalize_audio_on_export` is read at
  `waveform/commands.rs:447`.

## Risk register

| Risk | Mitigation | AC catching regression |
|------|------------|------------------------|
| A future contributor adds a second `blocks_to_ass` call for the sidecar, drifting the two documents | Enforce single-site via the `check-ass-generator-singleton.ps1` script gated on AC-003-a, plus a unit test | AC-003-a |
| Sidecar written for audio-only exports (ASS references non-existent frame size) | Explicit gate on `has_video && !export_format.is_audio_only()` inside `build_export_ass_doc` | AC-004-d |
| i18n key added only to `en/translation.json` | `scripts/check-translations.ts` gate; `i18n-pruning` skill on PR | AC-002-b |
| Sidecar and burn temp diverge because of trailing-newline or encoding differences | The same `String` value is handed to `std::fs::write` for both paths; unit test compares returned `String` to file bytes | AC-003-a + AC-003-b |
| 800-line cap breach in `commands/waveform/commands.rs` | Refactor into `export_captions.rs` sibling module if needed | Manual code-review / line-count gate |
| Stray `.burn_captions.ass` temp left on disk in the sidecar-off path | Existing cleanup (if any) unchanged; verify in AC-004-a | AC-004-a |
