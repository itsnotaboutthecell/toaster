# Blueprint: remove history and legacy

## Architecture decisions

- **R-001 — Hard delete, not feature-flag.** The history surface is
  removed from the build, not gated. There is no realistic future in
  which the transcript-first editor wants the dictation-era saved-
  transcription list back; gating just defers the deletion. Pattern
  followed: prior Handy-era prunes captured in
  `.github/skills/handy-legacy-pruning/SKILL.md` (delete; do not gate).

- **R-002 — Delete the Advanced panel; relocate survivors to
  single-purpose homes.** The Advanced panel is a grab-bag (model
  unload + experimental + word lists + caption + history limit +
  acceleration). Splitting it into focused panels matches the
  transcript-first editor IA. Pattern followed: existing single-
  purpose panels at `src/components/settings/models/ModelsSettings.tsx`
  and `src/components/settings/about/AboutSettings.tsx`.

- **R-003 — Re-grep before each per-file delete.** The PRD lists
  candidates; the executor must re-run the consumer-grep at deletion
  time and abort the delete if a non-Advanced consumer surfaces. This
  prevents accidental deletion of a file silently re-used elsewhere.

- **R-005 — Locale parity is a CI gate.** `scripts/check-translations.ts`
  already runs in CI; this feature only requires the deletion be
  symmetric across all 20 locales. No new tooling.

- **R-006 — Live-app QC is mandatory and time-stamped.** Per AGENTS.md
  "Verified means the live app", the manual undo/redo cycle must be
  recorded in `journal.md` with timestamp + operator initials, not
  asserted by `cargo check` alone.

## Single-source-of-truth placement

This feature does not introduce new dual-path logic. It removes
existing surfaces that already lived in a single place. The
non-negotiable single-source rules from AGENTS.md ("backend authority
for keep-segments, time mapping, caption layout, filler lists") are
preserved trivially because none of those subsystems are touched.

The one transient SoT concern: `CaptionSettings` is briefly hosted by
`ModelsSettings` until the parallel `caption-settings-preview`
feature lands its permanent home. The temporary host must not
duplicate caption-state logic; it imports `CaptionSettings` and
renders it as-is.

## Data flow

Removed flows:

- `frontend/HistorySettings.tsx`
  -> `commands.getHistoryEntries / toggleHistoryEntrySaved /
       getAudioFilePath / deleteHistoryEntry /
       retryHistoryEntryTranscription / updateHistoryLimit /
       updateRecordingRetentionPeriod`
  -> `commands::history::*`
  -> `managers::history::HistoryManager`
  -> `app_data_dir/history.db`
- `events.historyUpdatePayload` (from
  `managers::history::HistoryUpdatePayload`) -> dropped.

Preserved flows (untouched):

- `EditorView` -> `commands::editor::*` -> `managers::editor::*`
  (in-editor undo/redo lives here; this feature does not touch it).
- All splice / export / transcription paths.

## Migration / compatibility

- **User settings JSON.** `history_limit` and
  `recording_retention_period` may exist in saved settings on user
  disk. Serde must default-on-missing for surviving fields and ignore
  unknown keys (already the project pattern; verify by reading
  `settings/types.rs` for `#[serde(default = ...)]` usage and any
  top-level `#[serde(deny_unknown_fields)]` — if present, must be
  removed for the deleted fields).
- **Database file.** Existing installs keep a populated
  `app_data_dir/history.db` after upgrade. Harmless. A future
  housekeeping pass may delete it; out of scope here.
- **Tauri command surface.** Frontend will receive command-not-found
  errors if any stale UI fragment calls the removed commands. The
  bindings regeneration + TypeScript type-check (`tsc --noEmit` /
  `npm run build`) will catch this at build time.
